# 📌 LeetCode 3548 – Equal Sum Grid Partition II

## 🧩 Problem Statement

3548. Equal Sum Grid Partition II
You are given an m x n matrix grid of positive integers. Your task is to determine if it is possible to make either one horizontal or one vertical cut on the grid such that:

Each of the two resulting sections formed by the cut is non-empty.
The sum of elements in both sections is equal, or can be made equal by discounting at most one single cell in total (from either section).
If a cell is discounted, the rest of the section must remain connected.
Return true if such a partition exists; otherwise, return false.

Note: A section is connected if every cell in it can be reached from any other cell by moving up, down, left, or right through other cells in the section.

 

Example 1:

Input: grid = [[1,4],[2,3]]

Output: true

Explanation:
A horizontal cut after the first row gives sums 1 + 4 = 5 and 2 + 3 = 5, which are equal. Thus, the answer is true.
Example 2:

Input: grid = [[1,2],[3,4]]

Output: true

Explanation:
A vertical cut after the first column gives sums 1 + 3 = 4 and 2 + 4 = 6.
By discounting 2 from the right section (6 - 2 = 4), both sections have equal sums and remain connected. Thus, the answer is true.
Example 3:

Input: grid = [[1,2,4],[2,3,5]]

Output: false

Explanation:
A horizontal cut after the first row gives 1 + 2 + 4 = 7 and 2 + 3 + 5 = 10.
By discounting 3 from the bottom section (10 - 3 = 7), both sections have equal sums, but they do not remain connected as it splits the bottom section into two parts ([2] and [5]). Thus, the answer is false.
Example 4:

Input: grid = [[4,1,8],[3,2,6]]

Output: false

Explanation:
No valid cut exists, so the answer is false.

 

Constraints:

1 <= m == grid.length <= 105
1 <= n == grid[i].length <= 105
2 <= m * n <= 105
1 <= grid[i][j] <= 105

## 💡 Approach

### 🔹 Key Idea

We simulate splitting the grid in all possible ways and check:

1. If the sums are equal → ✅ valid partition
2. If not, check whether removing **one element** can balance the difference

---

### 🔹 Steps

1. **Compute Total Sum**

   * Calculate the total sum of all elements in the grid.

2. **Try Horizontal Cuts**

   * Iterate row by row.
   * Maintain:

     * `upperSum` → sum of top part
     * `lowerSum` → remaining sum
   * Compute:

     ```
     difference = upperSum - lowerSum
     ```

3. **Check Conditions**

   * ✅ Perfect split → `difference == 0`
   * ✅ Remove boundary element:

     * Top-left
     * Top-right
     * Current row left
   * ✅ Remove any previously seen element

4. **Reverse Rows**

   * Try cuts from bottom to top.

5. **Transpose Grid**

   * Convert columns → rows to reuse same logic for vertical cuts.

6. **Repeat Steps**

   * Apply same logic on transposed grid.

## ⚙️ Java Implementation

```java
import java.util.*;

class Solution {
    private long totalSum = 0;

    public boolean canPartitionGrid(int[][] grid) {
        int rows = grid.length;
        int cols = grid[0].length;

        // Step 1: Calculate total sum
        for (int[] row : grid) {
            for (int val : row) {
                totalSum += val;
            }
        }

        // Step 2: Try horizontal cuts
        if (isValidCut(grid)) return true;

        // Step 3: Reverse rows and try again
        reverseRows(grid);
        if (isValidCut(grid)) return true;

        // Restore grid
        reverseRows(grid);

        // Step 4: Transpose for vertical cuts
        int[][] transposed = transpose(grid);
        if (isValidCut(transposed)) return true;

        // Step 5: Reverse transposed and check again
        reverseRows(transposed);
        return isValidCut(transposed);
    }

    private boolean isValidCut(int[][] grid) {
        int rows = grid.length;
        int cols = grid[0].length;
        long upperSum = 0;
        Set<Long> seenValues = new HashSet<>();

        for (int r = 0; r < rows - 1; r++) {
            for (int c = 0; c < cols; c++) {
                upperSum += grid[r][c];
                seenValues.add((long) grid[r][c]);
            }

            long lowerSum = totalSum - upperSum;
            long difference = upperSum - lowerSum;

            if (difference == 0) return true;

            if (difference == grid[0][0] ||
                difference == grid[0][cols - 1] ||
                difference == grid[r][0]) {
                return true;
            }

            if (r > 0 && cols > 1 && seenValues.contains(difference)) {
                return true;
            }
        }
        return false;
    }

    private void reverseRows(int[][] grid) {
        int top = 0, bottom = grid.length - 1;
        while (top < bottom) {
            int[] temp = grid[top];
            grid[top] = grid[bottom];
            grid[bottom] = temp;
            top++;
            bottom--;
        }
    }

    private int[][] transpose(int[][] grid) {
        int rows = grid.length;
        int cols = grid[0].length;
        int[][] result = new int[cols][rows];

        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                result[j][i] = grid[i][j];
            }
        }
        return result;
    }
}
```

## 🐍 Python Implementation

```python
class Solution:
    def canPartitionGrid(self, grid):
        self.totalSum = 0
        rows, cols = len(grid), len(grid[0])

        # Step 1: Calculate total sum
        for row in grid:
            for val in row:
                self.totalSum += val

        # Step 2: Try horizontal cuts
        if self.isValidCut(grid):
            return True

        # Step 3: Reverse rows and try again
        self.reverseRows(grid)
        if self.isValidCut(grid):
            return True

        # Restore original grid
        self.reverseRows(grid)

        # Step 4: Transpose for vertical cuts
        transposed = self.transpose(grid)
        if self.isValidCut(transposed):
            return True

        # Step 5: Reverse transposed and try again
        self.reverseRows(transposed)
        return self.isValidCut(transposed)

    def isValidCut(self, grid):
        rows, cols = len(grid), len(grid[0])
        upperSum = 0
        seenValues = set()

        for r in range(rows - 1):
            for c in range(cols):
                upperSum += grid[r][c]
                seenValues.add(grid[r][c])

            lowerSum = self.totalSum - upperSum
            difference = upperSum - lowerSum

            if difference == 0:
                return True

            if (difference == grid[0][0] or
                difference == grid[0][cols - 1] or
                difference == grid[r][0]):
                return True

            if r > 0 and cols > 1 and difference in seenValues:
                return True

        return False

    def reverseRows(self, grid):
        grid.reverse()

    def transpose(self, grid):
        rows, cols = len(grid), len(grid[0])
        return [[grid[i][j] for i in range(rows)] for j in range(cols)]
```

### Time Complexity

### Space Complexity

* **O(n)** for `seenValues`
* **O(m × n)** for transpose
