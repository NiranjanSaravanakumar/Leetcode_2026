
# 🧩 LeetCode 3546 – Can Partition Grid

## 📌 Problem Statement

3546. Equal Sum Grid Partition I
You are given an m x n matrix grid of positive integers. Your task is to determine if it is possible to make either one horizontal or one vertical cut on the grid such that:

Each of the two resulting sections formed by the cut is non-empty.
The sum of the elements in both sections is equal.
Return true if such a partition exists; otherwise return false.

 

Example 1:

Input: grid = [[1,4],[2,3]]

Output: true

Explanation:



A horizontal cut between row 0 and row 1 results in two non-empty sections, each with a sum of 5. Thus, the answer is true.

Example 2:

Input: grid = [[1,3],[2,4]]

Output: false

Explanation:

No horizontal or vertical cut results in two non-empty sections with equal sums. Thus, the answer is false.

 

Constraints:

1 <= m == grid.length <= 105
1 <= n == grid[i].length <= 105
2 <= m * n <= 105
1 <= grid[i][j] <= 105
---

## 🧠 Intuition

* First, compute the **total sum** of all elements.
* If the total sum is **odd**, we can never split it into two equal parts.
* Otherwise, target becomes:

  ```
  target = totalSum / 2
  ```

Now:

* Try splitting **row-wise** → accumulate row sums
* Try splitting **column-wise** → accumulate column sums
* If at any point sum == target → ✅ valid partition

---

## ⚙️ Approach 1 (Brute Force with Running Sum)

### 🚀 Steps:

1. Compute `totalSum`
2. Traverse rows:

   * Keep adding row sums
   * Check if `rowSum * 2 == totalSum`
3. Traverse columns:

   * Keep adding column sums
   * Check if `colSum * 2 == totalSum`
4. Ensure partition is valid:

   * Stop before last row/column (`m-1`, `n-1`)

---

## 💻 Java Solution

```java
class Solution {
    public boolean canPartitionGrid(int[][] grid) {
        long totalSum = 0;
        long rowSum = 0;
        long colSum = 0;

        int m = grid.length;
        int n = grid[0].length;

        // Step 1: Calculate total sum
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                totalSum += grid[i][j];
            }
        }

        // Step 2: Row-wise partition
        for (int i = 0; i < m - 1; i++) {
            for (int j = 0; j < n; j++) {
                rowSum += grid[i][j];
            }
            if (rowSum * 2 == totalSum)
                return true;
        }

        // Step 3: Column-wise partition
        for (int j = 0; j < n - 1; j++) {
            for (int i = 0; i < m; i++) {
                colSum += grid[i][j];
            }
            if (colSum * 2 == totalSum)
                return true;
        }

        return false;
    }
}
```

## 🐍 Python 3 Solution

```python
class Solution:
    def canPartitionGrid(self, grid):
        total_sum = 0
        row_sum = 0
        col_sum = 0

        m = len(grid)
        n = len(grid[0])

        # Step 1: Total sum
        for i in range(m):
            for j in range(n):
                total_sum += grid[i][j]

        # Step 2: Row partition
        for i in range(m - 1):
            for j in range(n):
                row_sum += grid[i][j]
            if row_sum * 2 == total_sum:
                return True

        # Step 3: Column partition
        for j in range(n - 1):
            for i in range(m):
                col_sum += grid[i][j]
            if col_sum * 2 == total_sum:
                return True

        return False
```

### Input:

```
grid = [
  [1, 2, 3],
  [3, 2, 1]
]
```

```
true
```

### Explanation:

* Total sum = 12
* After first row → sum = 6
* Remaining = 6
  ✅ Valid horizontal partition

---

## ⚠️ Edge Cases

* Grid with only **1 row or 1 column**
* Total sum is **odd**
* Partition only valid if **both parts are non-empty**
