1878. Get Biggest Three Rhombus Sums in a Grid

You are given an m x n integer matrix grid​​​.

A rhombus sum is the sum of the elements that form the border of a regular rhombus shape in grid​​​. The rhombus must have the shape of a square rotated 45 degrees with each of the corners centered in a grid cell. Below is an image of four valid rhombus shapes with the corresponding colored cells that should be included in each rhombus sum:


Note that the rhombus can have an area of 0, which is depicted by the purple rhombus in the bottom right corner.

Return the biggest three distinct rhombus sums in the grid in descending order. If there are less than three distinct values, return all of them.

 

Example 1:


Input: grid = [[3,4,5,1,3],[3,3,4,2,3],[20,30,200,40,10],[1,5,5,4,1],[4,3,2,2,5]]
Output: [228,216,211]
Explanation: The rhombus shapes for the three biggest distinct rhombus sums are depicted above.
- Blue: 20 + 3 + 200 + 5 = 228
- Red: 200 + 2 + 10 + 4 = 216
- Green: 5 + 200 + 4 + 2 = 211
Example 2:


Input: grid = [[1,2,3],[4,5,6],[7,8,9]]
Output: [20,9,8]
Explanation: The rhombus shapes for the three biggest distinct rhombus sums are depicted above.
- Blue: 4 + 2 + 6 + 8 = 20
- Red: 9 (area 0 rhombus in the bottom right corner)
- Green: 8 (area 0 rhombus in the bottom middle)
Example 3:

Input: grid = [[7,7,7]]
Output: [7]
Explanation: All three possible rhombus sums are the same, so return [7].
 

Constraints:

m == grid.length
n == grid[i].length
1 <= m, n <= 50
1 <= grid[i][j] <= 105


🧠 Approach

We need to compute the border sum of all possible rhombuses in the grid.

A rhombus is defined by:

A center point (r, c)

A radius k

The four corners are:

      (r-k, c)
         ▲
         │
(r, c-k) ◄──► (r, c+k)
         │
         ▼
      (r+k, c)

For every cell (r, c):

Treat it as a center

Expand rhombus size k

Ensure the rhombus stays inside the grid

Compute the border sum

We store sums in a TreeSet to keep them:

sorted

unique

Finally, we return the largest three values.

⚡ Optimized Strategy (Diagonal Prefix Sum)

Instead of iterating through each border cell repeatedly, we use diagonal prefix sums.

We maintain two prefix arrays:

Prefix Array	Direction
diag1	top-left → bottom-right (↘)
diag2	top-right → bottom-left (↙)

This allows us to compute diagonal segments in O(1) time.

So the rhombus border sum can be computed using 4 diagonal segments.

⏱ Complexity Analysis
Time Complexity
O(m × n × min(m,n))
Space Complexity
O(m × n)

~~~~~~

Java 
class Solution {
    public int[] getBiggestThree(int[][] grid) {

        int m = grid.length, n = grid[0].length;

        int[][] diag1 = new int[m+1][n+1];
        int[][] diag2 = new int[m+1][n+2];

        for(int i=0;i<m;i++){
            for(int j=0;j<n;j++){
                diag1[i+1][j+1] = grid[i][j] + diag1[i][j];
                diag2[i+1][j] = grid[i][j] + diag2[i][j+1];
            }
        }

        TreeSet<Integer> set = new TreeSet<>();

        for(int r=0;r<m;r++){
            for(int c=0;c<n;c++){

                set.add(grid[r][c]);

                int k=1;

                while(r-k>=0 && r+k<m && c-k>=0 && c+k<n){

                    int topRight = diag1[r+1][c+k+1] - diag1[r-k][c];
                    int rightBottom = diag2[r+k+1][c] - diag2[r][c+k+1];
                    int bottomLeft = diag1[r+k+1][c+1] - diag1[r][c-k];
                    int leftTop = diag2[r+1][c-k] - diag2[r-k][c+1];

                    int total = topRight + rightBottom + bottomLeft + leftTop;

                    total -= grid[r-k][c];
                    total -= grid[r][c+k];
                    total -= grid[r+k][c];
                    total -= grid[r][c-k];

                    set.add(total);
                    k++;
                }
            }
        }

        int size = Math.min(3,set.size());
        int[] ans = new int[size];

        for(int i=0;i<size;i++){
            ans[i] = set.pollLast();
        }

        return ans;
    }
}

Python3 Version
from typing import List

class Solution:
    def getBiggestThree(self, grid: List[List[int]]) -> List[int]:
        m, n = len(grid), len(grid[0])
        s = set()

        for r in range(m):
            for c in range(n):
                # size 0 rhombus (single cell)
                s.add(grid[r][c])

                k = 1
                while r-k >= 0 and r+k < m and c-k >= 0 and c+k < n:
                    total = 0
                    x, y = r-k, c

                    # top -> right
                    for i in range(k):
                        total += grid[x+i][y+i]

                    # right -> bottom
                    for i in range(k):
                        total += grid[x+k+i][y+k-i]

                    # bottom -> left
                    for i in range(k):
                        total += grid[x+2*k-i][y-i]

                    # left -> top
                    for i in range(k):
                        total += grid[x+k-i][y-k+i]

                    s.add(total)
                    k += 1

        res = sorted(s, reverse=True)
        return res[:3]

📌 Key Points

Rhombus borders consist of 4 diagonals

Using diagonal prefix sums avoids repeated traversal

TreeSet ensures unique sorted results

Only the largest 3 values are returned
