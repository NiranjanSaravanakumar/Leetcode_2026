# 🔗 1722. Minimize Hamming Distance After Swap Operations

## 🧩 Problem Statement

1722. Minimize Hamming Distance After Swap Operations

You are given two integer arrays, source and target, both of length n. You are also given an array allowedSwaps where each allowedSwaps[i] = [ai, bi] indicates that you are allowed to swap the elements at index ai and index bi (0-indexed) of array source. Note that you can swap elements at a specific pair of indices multiple times and in any order.

The Hamming distance of two arrays of the same length, source and target, is the number of positions where the elements are different. Formally, it is the number of indices i for 0 <= i <= n-1 where source[i] != target[i] (0-indexed).

Return the minimum Hamming distance of source and target after performing any amount of swap operations on array source.

 

Example 1:
Input: source = [1,2,3,4], target = [2,1,4,5], allowedSwaps = [[0,1],[2,3]]
Output: 1
Explanation: source can be transformed the following way:
- Swap indices 0 and 1: source = [2,1,3,4]
- Swap indices 2 and 3: source = [2,1,4,3]
The Hamming distance of source and target is 1 as they differ in 1 position: index 3.

Example 2:
Input: source = [1,2,3,4], target = [1,3,2,4], allowedSwaps = []
Output: 2
Explanation: There are no allowed swaps.
The Hamming distance of source and target is 2 as they differ in 2 positions: index 1 and index 2.

Example 3:
Input: source = [5,1,2,4,3], target = [1,5,4,2,3], allowedSwaps = [[0,4],[4,2],[1,3],[1,4]]
Output: 0
 

Constraints:

n == source.length == target.length
1 <= n <= 105
1 <= source[i], target[i] <= 105
0 <= allowedSwaps.length <= 105
allowedSwaps[i].length == 2
0 <= ai, bi <= n - 1
ai != bi


---

## 💡 Key Insight

* If two indices are connected through swaps, they form a **connected component**.
* Within a component, you can **rearrange elements freely**.
* So, instead of swapping step-by-step:

  * Treat each connected component independently.
  * Match as many elements as possible between `source` and `target`.

---

## 🚀 Approach

### 1️⃣ Build Connected Components (Union-Find)

* Use **Disjoint Set (Union-Find)** to group indices that can be swapped.

### 2️⃣ Group Indices

* For each index, find its root parent.
* Group all indices with the same root.

### 3️⃣ Count Frequencies

For each group:

* Count frequency of elements in `source`.
* Try to match elements from `target`.

### 4️⃣ Compute Hamming Distance

* If a `target` value is not available in the group → mismatch.

---

## 🧠 Algorithm

```
1. Initialize Union-Find
2. Union all allowed swaps
3. Group indices by root
4. For each group:
    a. Count elements in source
    b. Match with target
    c. Count unmatched elements
5. Return total mismatches
```

---

## ⏱️ Complexity

| Operation           | Complexity |
| ------------------- | ---------- |
| Union-Find          | O(n α(n))  |
| Grouping            | O(n)       |
| Counting & Matching | O(n)       |
| **Total**           | **O(n)**   |

👉 `α(n)` = Inverse Ackermann function (almost constant)

## 🧾 Java Implementation

```java
import java.util.*;

class Solution {
    public int minimumHammingDistance(int[] source, int[] target, int[][] allowedSwaps) {
        int n = source.length;
        UnionFind uf = new UnionFind(n);

        // Step 1: Union all swaps
        for (int[] swap : allowedSwaps) {
            uf.union(swap[0], swap[1]);
        }

        // Step 2: Group indices
        Map<Integer, List<Integer>> groups = new HashMap<>();
        for (int i = 0; i < n; i++) {
            int root = uf.find(i);
            groups.computeIfAbsent(root, k -> new ArrayList<>()).add(i);
        }

        int hamming = 0;

        // Step 3: Process groups
        for (List<Integer> group : groups.values()) {
            Map<Integer, Integer> count = new HashMap<>();

            for (int idx : group) {
                count.put(source[idx], count.getOrDefault(source[idx], 0) + 1);
            }

            for (int idx : group) {
                int val = target[idx];
                if (count.getOrDefault(val, 0) > 0) {
                    count.put(val, count.get(val) - 1);
                } else {
                    hamming++;
                }
            }
        }

        return hamming;
    }

    class UnionFind {
        int[] parent;

        UnionFind(int n) {
            parent = new int[n];
            for (int i = 0; i < n; i++) parent[i] = i;
        }

        int find(int x) {
            if (parent[x] != x)
                parent[x] = find(parent[x]); // Path Compression
            return parent[x];
        }

        void union(int a, int b) {
            int pa = find(a);
            int pb = find(b);
            if (pa != pb) {
                parent[pa] = pb;
            }
        }
    }
}
```

## 🐍 Python Implementation

```python
from collections import defaultdict, Counter
from typing import List

class Solution:
    def minimumHammingDistance(self, source: List[int], target: List[int], allowedSwaps: List[List[int]]) -> int:
        n = len(source)

        parent = list(range(n))

        def find(x):
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]

        def union(a, b):
            pa, pb = find(a), find(b)
            if pa != pb:
                parent[pa] = pb

        # Step 1: Union
        for a, b in allowedSwaps:
            union(a, b)

        # Step 2: Group indices
        groups = defaultdict(list)
        for i in range(n):
            groups[find(i)].append(i)

        hamming = 0

        # Step 3: Process groups
        for group in groups.values():
            count = Counter()

            for idx in group:
                count[source[idx]] += 1

            for idx in group:
                if count[target[idx]] > 0:
                    count[target[idx]] -= 1
                else:
                    hamming += 1

        return hamming
```
