#LeetCode Problem: 3661

# 📌 Maximum Walls Destroyed by Robots
> 🟣 Difficulty: Hard
> 🏷️ Topics: Greedy, Binary Search, Dynamic Programming, Sorting

There is an endless straight line populated with some robots and walls. You are given integer arrays robots, distance, and walls:
robots[i] is the position of the ith robot.
distance[i] is the maximum distance the ith robot's bullet can travel.
walls[j] is the position of the jth wall.
Every robot has one bullet that can either fire to the left or the right at most distance[i] meters.

A bullet destroys every wall in its path that lies within its range. Robots are fixed obstacles: if a bullet hits another robot before reaching a wall, it immediately stops at that robot and cannot continue.

Return the maximum number of unique walls that can be destroyed by the robots.

Notes:

A wall and a robot may share the same position; the wall can be destroyed by the robot at that position.
Robots are not destroyed by bullets.
 

Example 1:
Input: robots = [4], distance = [3], walls = [1,10]
Output: 1
Explanation:
robots[0] = 4 fires left with distance[0] = 3, covering [1, 4] and destroys walls[0] = 1.
Thus, the answer is 1.

Example 2:
Input: robots = [10,2], distance = [5,1], walls = [5,2,7]
Output: 3
Explanation:
robots[0] = 10 fires left with distance[0] = 5, covering [5, 10] and destroys walls[0] = 5 and walls[2] = 7.
robots[1] = 2 fires left with distance[1] = 1, covering [1, 2] and destroys walls[1] = 2.
Thus, the answer is 3.

Example 3:
Input: robots = [1,2], distance = [100,1], walls = [10]
Output: 0
Explanation:
In this example, only robots[0] can reach the wall, but its shot to the right is blocked by robots[1]; thus the answer is 0.

 

Constraints:

1 <= robots.length == distance.length <= 105
1 <= walls.length <= 105
1 <= robots[i], walls[j] <= 109
1 <= distance[i] <= 105
All values in robots are unique
All values in walls are uni
---

## 💡 Key Idea

We solve this using:

### 1. **Sorting**

* Sort `robots` and `walls` to process in order

### 2. **Binary Search**

* Efficiently count walls in a given range using:

  * `lowerBound`
  * `upperBound`

### 3. **Precomputation**

For each robot `i`:

* `left[i]` → walls destroyed if shooting LEFT
* `right[i]` → walls destroyed if shooting RIGHT
* `overlap[i]` → shared walls between robot `i-1` and `i`

---

## ⚡ Why Overlap Matters?

If:

* robot `i-1` shoots RIGHT
* robot `i` shoots LEFT

👉 Their ranges may overlap → **double counting**

So we:

* subtract previous contribution
* add only valid merged walls

---

## 🧮 DP Transition

We maintain:

* `takeLeft` → max walls if current robot shoots LEFT
* `takeRight` → max walls if current robot shoots RIGHT

### Transition:

```text
newLeft = max(
    takeLeft + left[i],
    takeRight - right[i-1] + min(left[i] + right[i-1], overlap[i])
)

newRight = max(
    takeLeft + right[i],
    takeRight + right[i]
)
```

---

## ⏱️ Complexity

| Operation     | Complexity |
| ------------- | ---------- |
| Sorting       | O(n log n) |
| Binary Search | O(n log m) |
| DP Traversal  | O(n)       |

👉 **Overall:** `O(n log n)`

---

## ✅ Java Solution

```java
class Solution {
    public int maxWalls(int[] robots, int[] distance, int[] walls) {
        int n = robots.length;

        int[] left = new int[n];
        int[] right = new int[n];
        int[] overlap = new int[n];

        Map<Integer, Integer> robotDist = new HashMap<>();
        for (int i = 0; i < n; i++) {
            robotDist.put(robots[i], distance[i]);
        }

        Arrays.sort(robots);
        Arrays.sort(walls);

        for (int i = 0; i < n; i++) {
            int pos = robots[i];
            int d = robotDist.get(pos);

            int rightStartIdx = upperBound(walls, pos);

            int leftLimit = (i > 0) ? Math.max(pos - d, robots[i - 1] + 1) : pos - d;
            int leftStartIdx = lowerBound(walls, leftLimit);
            left[i] = rightStartIdx - leftStartIdx;

            int rightLimit = (i < n - 1) ? Math.min(pos + d, robots[i + 1] - 1) : pos + d;
            int rightEndIdx = upperBound(walls, rightLimit);
            int rightStart = lowerBound(walls, pos);
            right[i] = rightEndIdx - rightStart;

            if (i > 0) {
                int prevStart = lowerBound(walls, robots[i - 1]);
                overlap[i] = rightStartIdx - prevStart;
            }
        }

        int takeLeft = left[0];
        int takeRight = right[0];

        for (int i = 1; i < n; i++) {
            int newLeft = Math.max(
                takeLeft + left[i],
                takeRight - right[i - 1] + Math.min(left[i] + right[i - 1], overlap[i])
            );

            int newRight = Math.max(
                takeLeft + right[i],
                takeRight + right[i]
            );

            takeLeft = newLeft;
            takeRight = newRight;
        }

        return Math.max(takeLeft, takeRight);
    }

    private int lowerBound(int[] arr, int target) {
        int l = 0, r = arr.length;
        while (l < r) {
            int mid = (l + r) / 2;
            if (arr[mid] < target) l = mid + 1;
            else r = mid;
        }
        return l;
    }

    private int upperBound(int[] arr, int target) {
        int l = 0, r = arr.length;
        while (l < r) {
            int mid = (l + r) / 2;
            if (arr[mid] <= target) l = mid + 1;
            else r = mid;
        }
        return l;
    }
}
```

## 🐍 Python Solution

```python
import bisect
from typing import List

class Solution:
    def maxWalls(self, robots: List[int], distance: List[int], walls: List[int]) -> int:
        n = len(robots)

        robotDist = {robots[i]: distance[i] for i in range(n)}

        robots.sort()
        walls.sort()

        left = [0] * n
        right = [0] * n
        overlap = [0] * n

        for i in range(n):
            pos = robots[i]
            d = robotDist[pos]

            rightStartIdx = bisect.bisect_right(walls, pos)

            leftLimit = max(pos - d, robots[i - 1] + 1) if i > 0 else pos - d
            leftStartIdx = bisect.bisect_left(walls, leftLimit)
            left[i] = rightStartIdx - leftStartIdx

            rightLimit = min(pos + d, robots[i + 1] - 1) if i < n - 1 else pos + d
            rightEndIdx = bisect.bisect_right(walls, rightLimit)
            rightStart = bisect.bisect_left(walls, pos)
            right[i] = rightEndIdx - rightStart

            if i > 0:
                prevStart = bisect.bisect_left(walls, robots[i - 1])
                overlap[i] = rightStartIdx - prevStart

        takeLeft = left[0]
        takeRight = right[0]

        for i in range(1, n):
            newLeft = max(
                takeLeft + left[i],
                takeRight - right[i - 1] + min(left[i] + right[i - 1], overlap[i])
            )

            newRight = max(
                takeLeft + right[i],
                takeRight + right[i]
            )

            takeLeft = newLeft
            takeRight = newRight

        return max(takeLeft, takeRight)
```
