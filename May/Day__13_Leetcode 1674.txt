# 1674. Minimum Moves to Make Array Complementary

## Problem Statement
1674. Minimum Moves to Make Array Complementary

You are given an integer array nums of even length n and an integer limit. In one move,
you can replace any integer from nums with another integer between 1 and limit, inclusive.

The array nums is complementary if for all indices i (0-indexed), nums[i] + nums[n - 1 - i] equals the same number. 
For example, the array [1,2,3,4] is complementary because for all indices i, nums[i] + nums[n - 1 - i] = 5.

Return the minimum number of moves required to make nums complementary.

 

Example 1:
Input: nums = [1,2,4,3], limit = 4
Output: 1
Explanation: In 1 move, you can change nums to [1,2,2,3] (underlined elements are changed).
nums[0] + nums[3] = 1 + 3 = 4.
nums[1] + nums[2] = 2 + 2 = 4.
nums[2] + nums[1] = 2 + 2 = 4.
nums[3] + nums[0] = 3 + 1 = 4.
Therefore, nums[i] + nums[n-1-i] = 4 for every i, so nums is complementary.

Example 2:
Input: nums = [1,2,2,1], limit = 2
Output: 2
Explanation: In 2 moves, you can change nums to [2,2,2,2]. You cannot change any number to 3 since 3 > limit.

Example 3:
Input: nums = [1,2,1,2], limit = 2
Output: 0
Explanation: nums is already complementary.
 

Constraints:

n == nums.length
2 <= n <= 10^5
1 <= nums[i] <= limit <= 10^5
n is even.

## Intuition

For every pair:

(a, b)

we try to determine how many moves are needed for every possible target sum.

Possible target sums range from:

2 → 2 * limit

For each pair:

- 2 moves are required by default
- 1 move is required for a certain range
- 0 moves are required for one exact sum

Instead of checking every sum individually, we use a **Difference Array** to efficiently apply range updates.

---

## Key Observation

For a pair `(a, b)`:

Let:

low = min(a, b)
high = max(a, b)
sum = a + b

### Case 1 → 2 Moves

Any target sum initially requires 2 moves.

---

### Case 2 → 1 Move

We can achieve sums in:

[low + 1, high + limit]

using only one move.

---

### Case 3 → 0 Moves

Exact sum:

a + b

requires zero moves.

---

## Difference Array Technique

We maintain an array `diff[]` where:

- Add at start of range
- Subtract after end of range

Finally, prefix sum gives total moves needed for every target sum.

This reduces complexity from:

O(n * limit)

to:

O(n + limit)

---

# Algorithm

For every pair:

1. Assume 2 moves for all sums
2. Reduce 1 move for achievable range
3. Reduce another 1 move for exact sum

Then:

- Build prefix sum
- Find minimum moves

---

# Java Solution

```java
class Solution {
    public int minMoves(int[] nums, int limit) {

        int n = nums.length;

        int[] diff = new int[2 * limit + 2];

        for (int i = 0; i < n / 2; i++) {

            int a = nums[i];
            int b = nums[n - 1 - i];

            int low = Math.min(a, b);
            int high = Math.max(a, b);

            int sum = a + b;

            // Assume 2 moves for all sums
            diff[2] += 2;
            diff[2 * limit + 1] -= 2;

            // Reduce to 1 move
            diff[low + 1] -= 1;
            diff[high + limit + 1] += 1;

            // Reduce to 0 move
            diff[sum] -= 1;
            diff[sum + 1] += 1;
        }

        int ans = Integer.MAX_VALUE;
        int curr = 0;

        for (int s = 2; s <= 2 * limit; s++) {
            curr += diff[s];
            ans = Math.min(ans, curr);
        }

        return ans;
    }
}
